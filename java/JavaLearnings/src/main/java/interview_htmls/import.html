import statement

important statement are at second level in java program hierarchy after package

import statement are required to access any class which is not available in current directory

importing which have same method can cause ambiguity. e. java.sql and java.util both have Date class, so it creates ambiguity

import is implicit and explicit

implicit import is when you use * at the end, which will import all classes, interfaces from the directory, not from the subdirectories

explicit is when you specify only a single class in an import and only import that single class

static import is when you import using keyword static, all the static members of the class are imported and can be used directly without the class name

explicit import is given first preference

import pack1.pack2.pack3.*;

this will import all classess inside package pack3 but not further subpackages

import pack1.pack2.pack3.ClassA;

this will import only class A

import static pack1.pack2.pack3.ClassA

this will do a static import of classA

if we dont use import, we can given fully qualified name of the class along with its package and use it without import

java.lang is by default imported in each and every java program as it comtains most important classes, like System, String, Wrapper Classes

static import has to be done by pattern import pack1.pack2.Class.* - means all static members are imported of given class
