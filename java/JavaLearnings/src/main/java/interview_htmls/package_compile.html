package in java

// to provide namespace, encapsulate related class, subpackages, interfaces together

// it prevents naming conflicts

// it provide accessibility control

// packages are names in reverse order of domain of site, as they will provide unique name

// com.intelligent

// packages are of 2 types, user defined and system defined

// any package we create is user defined

// package already provided by lang framework are system defined, java.lang, java.util, java.sql

// easier to track down classes and interfaces if packages are used as related classes interfaces will be grouped in packages

// Every class is part of some package.

// If no package is specified, the classes in the file goes into a special unnamed package (the same unnamed package for all files).

// All classes/interfaces in a file are part of the same package. Multiple files can specify the same package name.

// If package name is specified, the file must be in a subdirectory called name (i.e., the directory name must match the package name).

// We can access public classes in another (named) package using: package-name.class-name

// only 1 package statement is allowed in java file, and will be applicable for all classes interfaces in that file

// package statement comes at the top in java program hierarchy

package college.art.history

package college.science.cse


Compile program

// javac tool is used for converting source code into bytecode

// java tool is used for converting bytecode to machine code, i.e for execution

// jar is tool used to combine .class files in jar package and run them all



javac abc.java // compile a java file without any package statement

java abc // run the .class



javac -d . abc.java // compile a java file with package and create .class in same directory as package, . means in current directory

java pack1.pack2.abc // run the file created from above statement



javac -d folder1/folder2 abc.java // compile a java file with package statement and save .class in folder1/folder2/package statement

java -cp folder1.folder2 pack1.pack2.abc // run .class with classpath option, go to the specified folder and then give fully qualified class name
